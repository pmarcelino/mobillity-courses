Congratulations on Building the Pipeline

Well done

You took raw, multi-table mobility data — the kind that quietly breaks a naive analysis — and turned it into a clean, joined dataset you could question and trust. That is the unglamorous skill that separates a real analysis from a plausible-looking guess.

What You’ve Accomplished

Look back at what was raw input a module ago and is analysis-ready now.

  • You handled missing values with intent. You can detect missingness in every form — NaN, empty strings, sentinels like s/n — and choose to drop, fill, interpolate, or flag based on your analytical goal, telling the LLM the why so it writes the right code.
  • You fixed silent type errors and judged outliers. You audit, convert, and verify column types before they corrupt a result, and you apply the “event or error?” test to an extreme value instead of deleting it on sight.
  • You joined related tables correctly. You can read the foreign-key structure of a multi-table feed and hand the LLM the schemas and join keys it needs to write joins that link the right rows.
  • You explored for insight. You drive the analysis with domain questions, direct the LLM to write the code that answers each, and chain the full sequence — load, clean, join, explore — on a real GTFS dataset.

Real-World Impact

Hand a working analyst a fresh bike-share export or a city’s GTFS feed and the first hours go to exactly this: detecting what is missing, fixing what is mistyped, joining what is related, and asking the first real questions. You can now run that opening sequence yourself — and explain every decision in it.

Key insight

Trustworthy exploration is built on cleaning, not bolted on after it. The LLM writes the code at every step; the strategy, the joins, and the questions are yours — and that ownership is what makes the answer defensible.

What’s Next: Visualizing Results

Insight buried in a table convinces no one. The next module turns these findings into visuals — directing the LLM to build static, statistical, interactive, and map-based charts, and refining them through plain-language feedback until they read clearly.

Before You Continue

Make sure the core moves feel solid:

  • Can you detect missing values in all their forms and justify a handling strategy from your analytical goal?
  • Can you explain why a type error produces a wrong answer with no error message?
  • Could you give the LLM what it needs to join four related GTFS tables correctly?
  • Can you take a cleaned dataset and drive an exploration through a chain of domain questions?
