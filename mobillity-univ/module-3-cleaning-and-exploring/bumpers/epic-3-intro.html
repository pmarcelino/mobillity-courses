Welcome to the Module

Last module you loaded a dataset and inspected it — and met the uncomfortable truth that real mobility data is rarely ready to use. Missing coordinates, durations stored as text, usage spikes you cannot immediately explain. This module is where you turn raw, messy files into data you can trust, then explore it for what it has to say.

The method does not change: you direct the LLM in plain language, it writes the Python, you run it and read the result. What changes is the work — from inspecting data to preparing it, joining it, and questioning it.

What You’ll Learn

Across six short lectures, you will build a complete cleaning-to-exploration pipeline on real European mobility data.

  • Handle missing values with intent. Detect missingness in all its disguises — blanks, NaN, sentinel labels like s/n — and choose whether to drop, fill, interpolate, or flag based on the question you are answering, not a reflex.
  • Fix silent type errors and judge outliers. Audit and convert columns stored as the wrong type — the mistakes that produce wrong answers with no warning — and decide, case by case, whether an extreme value is a real event or a data error.
  • Join related tables correctly. Recognise the foreign-key structure linking GTFS routes, trips, stop_times, and stops, and give the LLM the schemas and join keys it needs to write joins that do not hallucinate.
  • Explore for insight. Ask domain-rooted questions, direct the LLM to write the exploratory code that answers them, and follow each answer to the next — then run the whole pipeline end-to-end on a real GTFS feed.

Why Clean Data Matters

Mobility data arrives relational and imperfect: a bike-share snapshot with empty coordinates, a city’s GTFS feed split across four linked files, departure times that run past 24:00. An analysis built on uncleaned data is not just rough — it is quietly wrong, and it looks exactly as convincing as a correct one. Cleaning is what earns you the right to trust what exploration tells you.

Remember

Cleaning is not the chore before the real work — it is what makes the real work trustworthy. The LLM writes every line of cleaning and exploration code; you own the strategy, the join, and the question.

Ready to Begin?

The module opens with the most common mess of all — missing values — and the small discipline that separates a quick deletion from a defensible one: telling the model why.
