Code That Runs Is Not Code That’s Right

Welcome to the Module

Everything you have built so far has trusted one thing: that the LLM’s code did what you asked. This module removes that assumption. Code can run perfectly, produce a clean chart, and still answer a different question than the one you posed — a wrong join, a wrong column, a misread number.

As models get better at generating analyses, generating stops being the hard part. Checking becomes the job. This module builds the habit.

What You’ll Learn

Across four short lectures, you will build three layers of verification and learn to combine them into a single audit.

  • Verify by hand, fast. Three quick habits — read the code’s logic, spot-check a few rows against the raw data, and sanity-check whether a number is even plausible — each under a minute, each catching a different kind of mistake.
  • Direct the LLM to write its own checks. Describe in plain language what should be true of an analysis, and have the LLM write sanity-check code that passes quietly or raises when something looks wrong — verification that runs again every time the data changes.
  • Use the LLM as an evaluator. Ask it to review its own code against your original question, and to propose fast checks that would convince you the answer is right — a fresh reviewer framing catches what the first pass missed.
  • Audit an analysis end-to-end. Combine all three layers into one systematic pass and record what you found in an audit summary — finding, method, result, action.

Why Evaluation Matters

In mobility work the numbers drive decisions — service frequency, network coverage, equity across districts. A plausible-but-wrong figure that looks authoritative is more dangerous than an obvious error, because no one thinks to question it. Evaluation is what stands between a confident output and a defensible one.

Remember

Be skeptical by default: assume the output is wrong until you have verified it. A clean run tells you the code executed — it tells you nothing about whether it answered your question.

Ready to Begin?

The module starts with the cheapest, most powerful tools you have: your own eyes and three quick checks you can run on any analysis, no new code required.
